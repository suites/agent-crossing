# Generative Agents: Interactive Simulacra of Human Behavior

## 1. INTRODUCTION (서론)

- 본 논문은 신뢰할 수 있는 인간 행동을 시뮬레이션하는 컴퓨팅 소프트웨어 에이전트인 '생성 에이전트(Generative Agents)'를 제안한다.
- 대규모 언어 모델(LLM)을 확장하여 에이전트의 과거 경험을 자연어로 기억 스트림(Memory Stream)에 기록하고, 이를 종합하여 더 높은 수준의 성찰(Reflection)을 생성하며, 행동을 계획하기 위해 기억을 동적으로 검색(Retrieval)하는 아키텍처를 도입했다.
- 에이전트들은 'The Sims'에서 영감을 받은 샌드박스 환경인 Smallville에서 25명이 서로 상호작용하며, 개별적인 행동뿐만 아니라 그룹 차원의 창발적 사회 행동(Emergent social behaviors)을 신뢰할 수 있는 수준으로 보여준다.

## 2. RELATED WORK (관련 연구)

- 과거 수십 년간 인간의 행동을 대리(Proxies)하는 에이전트를 구현하기 위해 규칙 기반 시스템부터 인지 아키텍처(Cognitive architectures), 강화학습 등이 연구되어 왔다.
- 기존의 대규모 언어 모델(LLM)은 단일 시점의 인간 행동을 시뮬레이션하는 데에는 능하지만, 장기적인 일관성을 보장하기에는 한계가 있다.
- 신뢰할 수 있는 에이전트를 구현하려면 현재 환경뿐만 아니라 방대한 과거 경험에 기반하여 행동해야 하므로, 동적으로 기억을 업데이트하고 적절히 검색해내는 새로운 에이전트 아키텍처가 필수적이다.

## 3. GENERATIVE AGENT BEHAVIOR AND INTERACTION (생성 에이전트의 행동 및 상호작용)

- 에이전트들은 샌드박스 내에서 행동을 취하고 자연어로 다른 에이전트들과 소통한다. 에이전트의 자연어 행동 묘사는 언어 모델을 통해 이모지(Emoji)로 변환되어 시각적으로 나타난다.
- 사용자는 특정 에이전트의 주변 사물 상태를 자연어로 변경하여 개입하거나, 에이전트의 '내면의 목소리'로서 직접 명령을 내려 행동을 통제할 수 있다.
- 시뮬레이션을 통해 정보 확산(Information diffusion), 관계 기억(Relationship memory), 그룹 활동 조정(Coordination)과 같은 복잡한 사회 현상이 자율적으로 창발(Emergent)됨을 확인했다. 예를 들어, 한 에이전트에게 발렌타인데이 파티를 열겠다는 초기 의도만 주입해도 정보가 스스로 퍼져 여러 에이전트가 함께 파티에 참석한다.

## 4. GENERATIVE AGENT ARCHITECTURE (생성 에이전트 아키텍처) - ★구현 핵심

- 본 아키텍처는 에이전트의 장기적 일관성을 유지하기 위해 **기억(Memory), 성찰(Reflection), 계획(Planning)**의 세 가지 주요 구성 요소로 이루어져 있다.

### 4.1 Memory and Retrieval (기억 및 검색)

- **메모리 스트림(Memory Stream):** 에이전트가 경험한 모든 관찰(Observations)을 자연어 설명, 생성 타임스탬프, 최근 접근 타임스탬프와 함께 기록하는 핵심 데이터베이스다.
- **검색(Retrieval):** 현재 상황에 가장 관련성 높은 기억들만 추출하여 LLM의 컨텍스트 윈도우에 제공하기 위한 스코어링 기능이다. 검색 점수는 다음 수식을 통해 계산된다.
  - $score = \alpha_{recency} \cdot recency + \alpha_{importance} \cdot importance + \alpha_{relevance} \cdot relevance$
- 계산식 세부 구성 요소:
  - **최근성(Recency):** 최근에 접근한 기억일수록 점수가 높다. 마지막 검색 시점부터 지난 게임 시간(시간당)을 기준으로 감쇠(Decay) 인자 $0.995$를 적용한 지수 감쇠 함수를 사용한다.
  - **중요성(Importance):** 일상적인 기억과 중요한 기억을 구분한다. 기억 객체 생성 시 LLM에게 해당 사건이 얼마나 중요한지 1에서 10 사이의 정수 점수로 평가하도록 프롬프트를 부여하여 산출한다.
  - **관련성(Relevance):** 현재 쿼리 메모리와 각 기억의 텍스트 설명 사이의 임베딩 벡터 코사인 유사도(Cosine similarity)를 통해 계산한다.
- 구현 시 모든 점수는 Min-Max 스케일링을 통해 $$ 범위로 정규화되며, 본 논문에서는 모든 $\alpha$ 가중치를 $1$로 설정했다.

### 4.2 Reflection (성찰)

- 에이전트가 단순한 관찰을 넘어 더 높은 수준의 추론을 가능하게 하는 추상적 생각이다.
- 최근 인지한 이벤트들의 중요성(Importance) 점수 합이 특정 임계값(본 연구에서는 150)을 초과할 때마다 주기적으로 실행된다.
- 가장 최근 기억 100개를 바탕으로 LLM에 핵심 질문 3가지를 생성하도록 요청하고, 이 질문들을 쿼리로 사용하여 기억을 검색한 뒤 통찰(Insight)을 추출한다. 도출된 성찰 결과도 메모리 스트림에 저장되어 성찰 트리(Reflection tree)를 형성한다.

### 4.3 Planning and Reacting (계획 및 반응)

- **계획(Planning):** 장기적인 시간의 흐름 속에서 일관성을 확보하기 위해 미래의 행동(위치, 시작 시간, 소요 시간)을 순서대로 생성한다. 하향식(Top-down) 방식을 취해 하루의 대략적인 계획(5~8개 묶음)을 먼저 세우고, 이를 1시간 단위, 다시 5~15분 단위의 상세 행동으로 재귀적으로 분해(Decompose)한다.
- **반응 및 계획 수정(Reacting and Updating Plans):** 환경에서 특정 관찰이 발생하면 LLM을 통해 기존 계획을 유지할지 반응할지 결정한다. 반응이 필요하다고 판단되면 상호작용이 일어난 시점부터 계획을 다시 생성한다. 에이전트 간의 대화(Dialogue) 역시 서로의 요약된 기억을 바탕으로 생성된다.

## 5. SANDBOX ENVIRONMENT IMPLEMENTATION (샌드박스 환경 구현)

- Phaser 웹 게임 프레임워크를 기반으로 환경을 구축했으며, 샌드박스 서버는 각 에이전트의 현재 위치, 행동, 사물 상태를 JSON 데이터 구조로 관리한다.
- 자연어 기반의 아키텍처와 시각적 샌드박스를 연결하기 위해 월드 환경을 부모-자식 트리 데이터 구조(예: 집-부엌-스토브)로 표현한다. 에이전트가 특정 위치로 이동할 때는 트리를 순회(Traverse)하여 리프(Leaf) 노드 목적지를 찾고, 게임 경로 알고리즘을 사용해 이동시킨다.

## 6. CONTROLLED EVALUATION (통제된 평가)

- 에이전트의 능력을 검증하기 위해 자기 인식, 기억, 계획, 반응, 성찰의 5개 범주에 대한 인터뷰 평가를 진행했다.
- 절제 연구(Ablation)를 수행한 결과, 기억, 계획, 성찰이 모두 포함된 전체 아키텍처(Full Architecture)가 가장 신뢰할 수 있는(Believable) 행동을 생성했다.
- 한계점으로는 에이전트가 필요한 기억을 제대로 검색하지 못하거나, 모델 지식을 바탕으로 실제 일어나지 않은 사실을 덧붙이는 환각(Hallucination) 현상이 관찰되었다.

## 7. END-TO-END EVALUATION (엔드투엔드 평가)

- 2일(게임 시간) 간의 전체 시뮬레이션을 통해 창발적 행동을 검증했다. 정보 확산, 관계 형성, 에이전트 간 조정을 확인했다.
- 관계 형성 측정에 사용된 네트워크 밀도(Network density) 계산식은 다음과 같다.
  - $\eta = 2 * |E| / (|V|(|V|-1))$ (여기서 $|V|$는 에이전트(정점)의 수, $|E|$는 형성된 관계(간선)의 수)
- 이 수치 평가 결과, 시뮬레이션 초기에 0.167이었던 네트워크 밀도가 시뮬레이션 종료 후 0.74로 유의미하게 증가했다.
- 다만, 언어 모델의 지시 미세조정(Instruction tuning)으로 인해 에이전트들이 서로 지나치게 예의를 차리거나 지나치게 협조적으로 행동하는 경향이 발견되었다.

## 8. DISCUSSION (논의)

- 본 연구의 생성 에이전트는 소셜 컴퓨팅 프로토타이핑, 메타버스, 인체 공학 설계(HCI) 모델 등 다양하게 응용될 수 있다.
- 다만 윤리적, 사회적 위험이 수반된다. 사용자가 에이전트와 부적절한 파라소셜(Parasocial) 관계를 맺거나, 악의적 활용으로 딥페이크 및 맞춤형 설득에 악용될 위험을 내포하고 있다.
- 이를 완화하기 위해 에이전트가 계산된 개체임을 명확히 밝히고, 입력 및 출력 로그를 유지하며, 실제 사용자 조사를 완전히 대체하기보다는 초기 설계 단계의 도구로 활용할 것을 권장한다.

## 9. CONCLUSION (결론)

- 본 논문은 에이전트의 일상을 종합적으로 기록, 검색, 성찰, 계획하는 아키텍처를 도입함으로써 인간의 복잡한 행동과 상호작용을 구현한 '생성 에이전트'를 성공적으로 제시했다.
